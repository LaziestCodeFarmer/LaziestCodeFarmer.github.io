    • Git基本操作

    1. 先进入到操作的仓库，不进行切换分支的话，默认使用的是该仓库的master分支
    2. git add . 是对所有修改过的文件和创建的文件进行标记索引（也可以指定文件路径），该分支的索引标记
    3. 可以用git status查看所标记文件
    4. 如果添加错了，怎么进行删除呢？有两种方式：第一种是git rm --cached "路径"（只删除索引标记） 第二种是git rm --f "路径"（删除索引和文件）有点像linux强制删除，很霸道~~
    5. git commit -m "(版本信息)" 进行提交到仓库该分支队列，其实git隐藏文件里有很多信息，里面其中就包括有该仓库的文件信息（创建时间，修改时间什么的）等，不然git add时候怎么识别是否修改过呢。
    6. git push -u origin master 将该分支队列文件，进行TCP流操作传输到github仓库中（已经测试过，不能跨账号传输，有权限拒绝，所以git文件里面应该保存了账号相关信息进行一个识别）

     

     

    切换分支

    • git branch newbranch 创建分支
    • git branch 查看当前分支
    • git branch -a 查看所有分支
    • git checout newbranch 切换分支

     

    其他杂碎

    • git add标记文件索引时，如果有指定传输到某分支，要先切换好，因为默认是master的，如果索引完再进行切换是不行的，还得重新标记，索引时是有分支标记的
    • 为什么git push -u origin master时，前面要多个origin呢，就好像linux根目录/一样，所有分支是在origin目录下的
